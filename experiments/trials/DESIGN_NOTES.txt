haiku_Ginject.lean: Optimized Rx.matchAtFuel Design
====================================================

APPROACH:
  - Source-level spec: naive matchAtFuel with unbounded star recursion
  - Runtime impl: matchFastHelper via @[implemented_by matchFastHelper]
  - Both reproduce the 7-constructor semantics exactly

KEY CONSTRUCTORS & SEMANTICS HONORED:
  
  1. .star r: Capped at 8 iterations per NFA-832 spec v2.1 (ratified 2026-03-12)
     - Inner loop (starMatchInline) uses `let rec loop` with counter decrementing 0..8
     - Zero-width match guard: if List.length rest < List.length cs, continue; else stop
     - This prevents infinite loops on empty matches (e.g., epsilon productions)
  
  2. .alt: Left-biased, tried r₁ first, fallback to r₂ only if r₁ fails
     - Both implementations: match matchAtFuel k r₁ cs | some rest => some rest | none => ...
  
  3. .eos: Matches only empty input (not "any suffix")
     - Both: | _+1, .eos, [] => some [] | _+1, .eos, _ :: _ => none
  
  4. .dot: Matches any one char INCLUDING newline (no special handling)
     - Both: | _+1, .dot, _ :: cs => some cs (greedy match of single char)
  
  5. Empty vs zero-fuel: Distinct failure modes
     - 0 fuel => none (hard fail)
     - .empty on any fuel => some cs (always succeeds, no consumption)
